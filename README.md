# Iterator
Итератор — это поведенческий паттерн проектирования, который даёт возможность последовательно обходить элементы составных объектов, не раскрывая их внутреннего представления
(из книги "Погружение в паттерны проектирования")

Обладает следующими характеристиками:
Тип: поведенческий.

Принцип работы:

Паттерн Iterator состоит из следующих компонентов:

Iterator  Определяет интерфейс для итерации по коллекции. Он предоставляет методы для перемещения по коллекции (moveNext) и получения текущего элемента (current).

Collection: Представляет коллекцию, которую необходимо обойти. Он предоставляет метод CreateIterator, который возвращает итератор для коллекции.

Concrete Iterator (Конкретный итератор): Реализует интерфейс Итератора для конкретной коллекции. Он знает внутреннюю структуру коллекции и управляет перемещением по ней.


Назначение: Iterator позволяет реализовать несколько вариантов обхода структуры данных без захламления самого класса коллекции.
Метафора: Экскурсия по городу, где сами достопримечательности города - коллекция. Есть следующие варианты их посещения: "пойти самому + потратить кучу времени + бесплатно"б "использовать виртуальный гид + потратить чуть меньше времени + платно" и "локальный гид + вообще почти без трат времени + платно*2". В каждом варианте мозг/навигатор/гид есть итератор, а мы, как код, можем выбрать один из них, отталкиваясь от условий решаемой задачи.
Потенциальные проблемы: Если указанное выше можно реализовать простым циклом, то зачем делать Iterator?
